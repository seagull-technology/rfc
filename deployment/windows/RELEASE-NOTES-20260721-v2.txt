RFC e-Permit staging release 20260721-v2
=========================================

This is a complete staging release built from the current main branch. It includes all
application and UAT work already present in the repository, plus the latest
government integration updates. It is not a partial API hotfix.

SANAD / SignFlow integration:

- Adds secure SANAD login using OAuth state validation and PKCE S256.
- Maps the verified SANAD national number to an existing RFC user account.
- Does not create accounts automatically; missing, pending, or inactive users
  are rejected according to the existing account rules.
- Adds SignFlow token, refresh, signing, introspection, identity, certificate,
  signature information, status, and logout API clients.
- Adds a super-admin SANAD registration-status diagnostic.
- Adds offline Bruno and Postman requests for both SignFlow API products.

MOHE integration:

- Uses the MOHE Standard Undergraduate Students LastSemester endpoint.
- Selects the current enrolled record when the provider returns both graduate
  and enrolled history for the same national number.
- Keeps graduate-only applicants ineligible for student registration.

Deployment notes:

- The deployment script preserves the government server .env, uploaded files,
  and database data.
- Configure both SignFlow API products and the provider-supplied browser
  authorization URL before testing SANAD login.
- MODEE must register SANAD_REDIRECT_URI exactly as configured on the server.
- Keep all client secrets in the server .env; none are included in this archive.
- No new database migration is required by this release.
