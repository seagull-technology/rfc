RFC GOVERNMENT SERVER UPGRADE CHECKLIST
=======================================

Before running the deployment:

1. Ask the database administrator to create a snapshot/backup of FSStgMDLDB.
2. Confirm C:\inetpub\rfc\.env exists and contains the current production secrets.
3. Reconcile the server .env against:

   deployment\windows\.env.production.example

   Keep every key only once. Preserve the existing APP_KEY, database credentials,
   GSB credentials, and seeded account credentials rather than copying placeholders.
   Confirm these non-secret values are present:

   APP_ENV=production
   APP_DEBUG=false
   APP_TIMEZONE=Asia/Amman
   GSB_ENABLED=true
   GSB_ENVIRONMENT=stg
   GSB_BASE_URL=https://api-gateway.stg.gsb.gov.jo:9443
   GSB_CREW_VERIFICATION_MINUTES=120
   GSB_MOHE_UNDERGRADUATE_ENABLED=true
   GSB_MOHE_UNDERGRADUATE_PATH=/porg-g2g/g2g/api/LastSemester
   GSB_MOHE_UNDERGRADUATE_METHOD=POST
   GSB_CCD_COMPANY_ENABLED=true
   GSB_CCD_COMPANY_PATH=/porg-g2g/g2g/api/companies/CompanybyNo/{nationalNo}
   GSB_CCD_COMPANY_METHOD=GET
   GSB_MIT_SERVICES_ENABLED=true
   GSB_MIT_SERVICES_PATH=/porg-g2g/g2g/api/Registry/getIndividualRegistry
   GSB_MIT_SERVICES_METHOD=POST
   GSB_CSPD_MASKED_ENABLED=true
   GSB_CSPD_MASKED_PATH=/porg-g2g/g2g/masked/api/person-info
   GSB_CSPD_MASKED_METHOD=GET
   GSB_CSPD_MASKED_PSD_PATH=/porg-g2g/g2g/masked/api/psd
   GSB_PSD_BASIC_INFO_ENABLED=true
   GSB_PSD_BASIC_INFO_PATH=/porg-g2g/g2g/token/psd
   GSB_PSD_BASIC_INFO_METHOD=POST
   GSB_SIGNFLOW_V2_ENABLED=true
   GSB_SIGNFLOW_V2_PATH=/porg-g2g/g2g/signflow/v2
   GSB_SIGNFLOW_V2_METHOD=POST
   GSB_SIGNFLOW_V2_OPEN_ENABLED=true
   GSB_SIGNFLOW_V2_OPEN_PATH=/porg-g2g/g2g/signflow/v2
   GSB_SIGNFLOW_V2_OPEN_METHOD=POST

   Keep GSB_SIGNFLOW_V2_ENABLED and GSB_SIGNFLOW_V2_OPEN_ENABLED false until
   MODEE supplies the browser authorization endpoint and registers the RFC
   callback URL. Then set:

   SANAD_AUTHORIZATION_URL=<provider URL>
   SANAD_REDIRECT_URI=https://<public host>/ar/login/sanad/callback
   SANAD_SCOPE=openid

4. Keep the real GSB_CLIENT_ID and GSB_CLIENT_SECRET values already stored in .env.
   Add the OAuth bearer supplied for PSD BASIC INFO Token as:

   GSB_PSD_BASIC_INFO_BEARER=<provider bearer token>

   Keep GSB_CSPD_TOKEN_ENABLED=false for the Personal Details Form. That token
   product returns the identity represented by its bearer and does not accept the
   form's arbitrary national number as an input.

   Store SANAD_CLIENT_ID and SANAD_CLIENT_SECRET separately when the SignFlow
   product subscription uses different IBM credentials.
5. Confirm APP_URL and SANAD_REDIRECT_URI use the final HTTPS public host. Until
   HTTPS is active, set SESSION_SECURE_COOKIE=false; switch it to true with HTTPS.
6. Confirm C: has enough free space for the new release and one full application backup.
7. Run PowerShell as Administrator.

Deployment command example:

  Set-ExecutionPolicy -Scope Process Bypass
  C:\Deploy\rfc-offline-release-20260722-v1\Deploy-RfcRelease.ps1 `
    -ArchivePath C:\Deploy\rfc-offline-release-20260722-v1\rfc-app.tar.gz

The script preserves .env and storage, applies migrations, refreshes permissions and
reference lookups without changing existing Super Admin/RFC passwords, swaps the
application, rebuilds caches, and checks http://localhost/ar/login.

Do not delete the generated C:\inetpub\rfc-backup-* directory until UAT is complete.
