RFC GOVERNMENT SERVER UPGRADE CHECKLIST
=======================================

Before running the deployment:

1. Ask the database administrator to create a snapshot/backup of FSStgMDLDB.
2. Confirm C:\inetpub\rfc\.env exists and contains the current production secrets.
3. Confirm these non-secret values are present in .env:

   APP_ENV=production
   APP_DEBUG=true
   APP_TIMEZONE=Asia/Amman
   GSB_ENABLED=true
   GSB_ENVIRONMENT=stg
   GSB_BASE_URL=https://api-gateway.stg.gsb.gov.jo:9443
   GSB_MOHE_SANAD_ENABLED=true
   GSB_MOHE_SANAD_PATH=/porg-g2g/g2g/newstandard/api/MoheStandard
   GSB_CCD_COMPANY_ENABLED=true
   GSB_CCD_COMPANY_PATH=/porg-g2g/g2g/api/companies/CompanybyNo/{nationalNo}
   GSB_CCD_COMPANY_METHOD=GET
   GSB_MIT_SERVICES_ENABLED=true
   GSB_MIT_SERVICES_PATH=/porg-g2g/g2g/api/Registry/getIndividualRegistry
   GSB_MIT_SERVICES_METHOD=POST

4. Keep the real GSB_CLIENT_ID and GSB_CLIENT_SECRET values already stored in .env.
5. Confirm C: has enough free space for the new release and one full application backup.
6. Run PowerShell as Administrator.

Deployment command example:

  Set-ExecutionPolicy -Scope Process Bypass
  C:\Deploy\rfc-offline-release-20260721-v1\Deploy-RfcRelease.ps1 `
    -ArchivePath C:\Deploy\rfc-offline-release-20260721-v1\rfc-app.tar.gz

The script preserves .env and storage, applies migrations, refreshes permissions and
reference lookups without changing existing Super Admin/RFC passwords, swaps the
application, rebuilds caches, and checks http://localhost/ar/login.

Do not delete the generated C:\inetpub\rfc-backup-* directory until UAT is complete.
